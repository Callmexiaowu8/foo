# macOS 菜单栏集成说明文档

## 概述

本文档详细说明倒计时提醒应用的菜单栏功能实现，包括架构设计、功能特性和性能指标。

## 功能特性

### 1. 菜单栏图标显示

#### 显示模式
- **动态倒计时模式**: 实时显示第一个活跃计时器的剩余时间（主要且唯一模式）

#### 菜单栏布局
- **水平导航栏设计**：操作按钮与标题并排显示
- **左侧**：标题 + 沙漏图标
- **右侧**：暂停/继续、取消、新建、打开主窗口按钮（有计时器时显示）
- **分隔线**：在操作按钮之间区分暂停/取消组与新建/打开组

#### 显示逻辑
- 无活跃计时器时：显示默认 timer 图标
- 单个活跃计时器：显示倒计时时间（如 "25:00"）
- 多个活跃计时器：显示第一个计时器时间 + 数量（如 "25:00 | 3"）

#### 主题适配
- 自动适配 macOS 深色/浅色模式
- 使用 SF Symbols 图标确保清晰度
- 图标使用 `isTemplate = true` 模式自动适配系统主题

### 2. 下拉菜单交互

#### 菜单结构
```
┌────────────────────────────────┐
│ ⏱️ 倒计时               [+]   │  ← 极简标题栏
├────────────────────────────────┤
│ ● 喝水                         │
│   24:56  [⏸] [↺] [✕]       │
│                                │
│ ● 休息                         │
│   45:30  [▶] [↺] [✕]        │
├────────────────────────────────┤
│ 2 个计时器  [全部暂停] 详情 退出 │  ← 底部状态栏
└────────────────────────────────┘
```

**计时器控制按钮说明**
- `[▶]` 开始：未运行的计时器
- `[⏸]` 暂停：运行中的计时器
- `[↺]` 重置：恢复至初始值
- `[✕]` 取消：终止并清除
- 状态指示圆点：🟢运行中 🟠已暂停 ⚪未启动

#### 交互特性
- 菜单响应时间 < 200ms
- 悬停效果实时反馈
- 支持键盘快捷键操作

### 3. 性能指标

#### 内存占用
- 菜单栏组件内存占用: < 5MB（目标 < 10MB）
- 定期检查内存使用，超过阈值时发出警告

#### CPU 使用率
- Idle 状态 CPU 使用率: < 1%（目标 < 5%）
- 使用节流控制，更新间隔 100ms

#### 响应时间
- 菜单打开响应时间: < 100ms（目标 < 200ms）
- 操作反馈延迟: < 50ms

## 集成步骤

### 1. 初始化配置

在应用启动时配置菜单栏管理器：

```swift
@main
struct CountdownReminderApp: App {
    @StateObject private var timerManager = TimerManager()
    
    var body: some Scene {
        WindowGroup {
            ContentView()
                .onAppear {
                    // 配置菜单栏
                    MenuBarManager.shared.configure(with: timerManager)
                }
        }
    }
}
```

### 2. 显示模式切换

```swift
// 切换到静态图标模式
MenuBarManager.shared.setDisplayMode(.staticIcon)

// 切换到动态倒计时模式
MenuBarManager.shared.setDisplayMode(.dynamicTimer)

// 切换到紧凑模式
MenuBarManager.shared.setDisplayMode(.compact)
```

### 3. 菜单交互

```swift
// 打开菜单
if let button = statusItem?.button {
    MenuBarManager.shared.openMenu(relativeTo: button)
}

// 关闭菜单
MenuBarManager.shared.closeMenu()

// 显示主窗口
MenuBarManager.shared.showMainWindow()
```

## 故障排除

### 常见问题

#### Q: 菜单栏图标不显示
A: 检查 `Info.plist` 中的 `LSUIElement` 设置，确保为 `false`

#### Q: 深色模式适配异常
A: 确认图标设置 `image.isTemplate = true`

#### Q: 菜单响应慢
A: 检查是否有耗时操作阻塞主线程，使用 Instruments 分析

#### Q: 内存占用过高
A: 检查是否有循环引用，使用 Memory Graph 调试

## 版本号规范

本文档采用语义化版本号 (Semantic Versioning) 规范，格式为 **vX.Y.Z**：

| 版本类型 | 格式 | 适用场景 | 示例 |
|----------|------|----------|------|
| **主版本号** | vX.0.0 | 重大架构重构、破坏性变更、不兼容的API修改 | v2.0.0 |
| **次版本号** | vX.Y.0 | 新功能添加、功能改进、较大优化（保持向后兼容） | v1.5.0 |
| **补丁版本** | vX.Y.Z | Bug修复、小幅优化、文档更新（不新增功能） | v1.4.1 |

### 版本号更新规则

1. **主版本号 (X)** 递增时，Y 和 Z 必须重置为 0
   - 适用于：项目结构重大调整、架构重写、破坏性变更

2. **次版本号 (Y)** 递增时，Z 必须重置为 0
   - 适用于：添加多个新功能、功能显著改进、大幅优化

3. **补丁版本 (Z)** 递增时，X 和 Y 保持不变
   - 适用于：Bug修复、小的改进、文档更新

### 版本示例

```
v1.0.0 - 初始版本发布
v1.0.1 - 修复一个小Bug
v1.1.0 - 新增提醒功能
v1.1.1 - 修复提醒功能的Bug

v2.0.1 - 修复架构重构后的问题
v2.1.0 - 新增快捷键功能
```

### 版本兼容性原则

- **主版本号变更**：可能不兼容旧版本数据或API
- **次版本号变更**：向后兼容，可能需要数据迁移
- **补丁版本变更**：完全向后兼容
